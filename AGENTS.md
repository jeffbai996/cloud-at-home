# Agent guidance

- Treat this repository as the durable workspace for `cloud-at-home` work.
- When an attachment arrives through a reduced API/tool route, describe that
  limitation as specific to the current turn. Do not claim that repository,
  shell, or write access has been permanently lost.
- Preserve or requeue unfinished implementation work for the normal Codex
  engine instead of presenting an unexecuted patch as completed.
- Keep public source free of private hostnames, URLs, credentials, media, and
  deployment-specific configuration. Run `npm run check:public` before commit.
